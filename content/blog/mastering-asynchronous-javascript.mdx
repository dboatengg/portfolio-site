JavaScript used to be infamous for *callback hell*.  
Then Promises arrived. And now, `async/await` has made asynchronous code readable again — if you use it right.

### 1. Understanding the Event Loop

Everything in JavaScript runs on a single thread.  
The trick is — that thread *doesn’t block*. The event loop lets async operations queue callbacks while your code keeps running.

```js
console.log("Start")

setTimeout(() => {
  console.log("Async operation complete")
}, 0)

console.log("End")
